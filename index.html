Minimum viable architecture

1

Approved SBOM ingestion

Accept supplier or OEM SBOMs only. Validate format and required fields before storage.

2

Structured SBOM repository

Store components, versions and dependencies in a queryable graph-like structure.

3

Aircraft mapping layer

Connect aircraft → subsystem → SBOM so software impact becomes real fleet impact.

4

Impact lookup engine

Input a component or vulnerability ID and return affected SBOMs, subsystems and aircraft.

5

Dashboard output

Show results clearly with dependency tree, affected assets and priority level.

Scope, value and constraints

System boundary

MVC

Includes: SBOM ingestion, storage, dependency tree, aircraft mapping and impact lookup.
Interfaces with: supplier/OEM SBOM exports and aircraft configuration records.
Does not do: real-time supplier APIs, Skywise integration, blockchain provenance, predictive AI, audit tooling or fleet heatmaps.

OEM

Value

Reduced integration risk and faster vulnerability triage.

Airlines

Value

Less downtime through clearer aircraft impact visibility.

MRO

Value

Faster diagnostics and targeted maintenance planning.

Regulators

Value

More transparent traceability from software issue to aircraft exposure.

Key constraints

Reality check

Data trust, supplier consistency, aviation certification boundaries, configuration accuracy, cybersecurity access control, and deployment cost/scalability.
